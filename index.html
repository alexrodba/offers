El proyecto

Una de las clínicas privadas más grandes de Europa en salud femenina

Dexeus Dona es una clínica especializada en ginecología, obstetricia y medicina de la reproducción en Barcelona. Han solicitado un portal digital para pacientes accesible desde cualquier navegador, con experiencia excelente y personalizada, integrando herramientas clínicas y administrativas existentes junto con nuevas funcionalidades.

Nota importante. La RFP no incluye el nivel de detalle necesario para una estimación con garantías. Recomendamos una fase previa de definición de 2-3 semanas antes de comprometerse al proyecto completo. Esta propuesta incluye los márgenes de riesgo apropiados.

Stack técnico

Arquitectura pensada para durar

Frontend SPA en Angular, API backend en Ruby on Rails. Monolito modular: más ágil que microservicios, más fácil de mantener.

🅰
Angular
Última versión estable. Standalone Components, Signals, lazy loading y accesibilidad web de serie.
💎
Ruby on Rails
Framework maduro con ecosistema estable. APIs REST. Usado por Shopify, GitHub y Stripe.
🎨
Tailwind CSS
Utilidades CSS para interfaces personalizadas, consistentes y rápidas de desarrollar.
🐘
PostgreSQL
Base de datos relacional principal. Robusta, probada y con soporte a largo plazo.
⚡
Redis
Caché de API y gestión de colas de procesos asíncronos en segundo plano.
🐳
Docker + CI/CD
Contenedores Docker, GitHub Actions, Prometheus + Grafana para monitorización.

Riesgos identificados

Transparencia total

Preferimos identificar los riesgos ahora que descubrirlos en producción. Para cada uno, tenemos un plan.

Alto
Integraciones no documentadas
Sin el listado completo de sistemas y su documentación, las estimaciones de la Fase 4 pueden desviarse significativamente.
MitigaciónHoras fijas para estudio. Revisión de estimación al inicio de la Fase 3. Posibilidad de fase extra o sacar funcionalidades de menor prioridad.
Alto
Sin acceso al modelo de datos actual
Sin poder estudiar la plataforma existente, la estimación de migración (40h) puede ser insuficiente.
MitigaciónHoras para estudio y plan de migración. Con acceso al modelo de datos, refinamos la estimación antes de ejecutar.
Medio
Retraso en Fases 1 y 2 (Polariza)
Factores como retrasos en User Research o más iteraciones de las previstas pueden impactar el inicio del desarrollo.
MitigaciónComunicación semanal con Polariza. Repriorización de funcionalidades o reducción de alcance si es necesario.
Medio
Bloqueos atribuibles al cliente
Aprobaciones tardías, permisos no disponibles o bajas de personal clave pueden generar parones.
MitigaciónCambio de prioridades, reorganización de fases o pausa del desarrollo sin facturación si los bloqueos lo permiten.
Bajo
Componentes fuera de Angular Material
Algunos componentes de UI pueden requerir desarrollo a medida si la librería no los cubre.
MitigaciónReutilización de componentes de proyectos anteriores. Desviación menor, compensable simplificando estilos.
